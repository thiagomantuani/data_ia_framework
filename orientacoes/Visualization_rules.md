# Visualização

## Dashboards devem conter

### Macro
- **KPIs** — métricas principais de negócio, sempre visíveis
- **Tendências** — evolução temporal de indicadores-chave
- **Variações percentuais** — comparativo com período anterior, meta ou benchmark

### Micro
- **Segmentações** — cortes por categoria, região, perfil, período
- **Comparativos** — grupo A vs grupo B, atual vs histórico, observado vs esperado
- **Drill-down** — navegação hierárquica do resumo ao detalhe

## Evitar
- **Excesso de gráficos** — mais de 5 visualizações numa mesma tela polui e dispersa
- **Gráficos sem contexto** — todo gráfico precisa de título, label de eixo, fonte e período
- **Poluição visual** — cores demais, efeitos 3D, sombras, gradientes — simplicidade transmite melhor

## Princípios
1. Cada visualização deve responder a **uma pergunta de negócio**
2. Dados agregados no macro; detalhados no micro
3. Código de cores consistente: verde = positivo, vermelho = negativo, cinza = neutro
4. Privacidade: nunca expor dados individuais (PII) em dashboards públicos
